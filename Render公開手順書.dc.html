スマホアプリ公開手順書（Render 版）

予定表を iPhone・Android の「ホーム画面アプリ」として使えるように、Render という無料サービスに公開します。 公開は最初の1回だけ。以降は各スマホで「ホーム画面に追加」するだけです。 データの保存先は今までどおり Google（GAS）で、そのまま同期されます。

用意するもの
  • お渡しする アプリ一式フォルダ（app）（この手順書と一緒にダウンロード）
  • メールアドレス（GitHub と Render の無料登録に使用）

※ Render はファイルを「GitHub」という置き場に置いてから公開する仕組みです。難しい操作はなく、すべてブラウザ上のドラッグ＆ドロップで完了します。

ステップ1　GitHub にファイルを置く

  1. github.com/signup で無料アカウントを作成します。
  2. ログイン後、右上「＋」▸ New repository。名前を puatto-app 等にし、Public のまま Create repository。
  3. 次の画面の 「uploading an existing file」 リンクをクリック。
  4. お渡しした app フォルダの中身（フォルダごとではなく中のファイル全部）を、画面にドラッグ＆ドロップ。
  5. 下の Commit changes ボタンを押します。これでファイルが置かれました。

ステップ2　Render で公開する

  1. render.com を開き、「Get Started」から GitHub アカウントで登録します。
  2. ダッシュボードで New ▸ Static Site を選びます。
  3. 先ほどの puatto-app リポジトリを選び Connect。
  4. 設定は次のようにします：
    ・Build Command：（空欄でOK）
    ・Publish Directory：. （半角ピリオド1つ）
  5. Create Static Site を押すと数分で公開されます。画面上部に https://puatto-app.onrender.com のような URL が出ます。これがアプリの住所です。

ステップ3　スマホでアプリにする

iPhone（Safari）
  1. Safari で公開URLを開く
  2. 下の 共有ボタン（□↑） をタップ
  3. 「ホーム画面に追加」を選ぶ
  4. アイコンが追加され、全画面で起動します
Android（Chrome）
  1. Chrome で公開URLを開く
  2. 右上の ︙（メニュー） をタップ
  3. 「アプリをインストール」または「ホーム画面に追加」
  4. アイコンが追加され、全画面で起動します

ステップ4　同期はすでに設定済み

データの保存先（Google の /exec URL）はアプリにあらかじめ組み込み済みです。各端末での貼り付けは不要で、アプリを開くと自動的につながります。 予定表画面の右下ピルが 「同期中（受信○件）」 になっていれば正常です。

※ 保存先を変えたいときだけ、右下ピルをタップして別の URL に変更できます。

💡
予定表を直したくなったら、GitHub の同じリポジトリでファイルを差し替えるだけで、Render が自動で更新します。URL・アイコンはそのままです。
